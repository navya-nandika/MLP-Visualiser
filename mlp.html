Definition
WHAT IS AN MLP?
Multilayer Perceptron — Definition
A Multilayer Perceptron (MLP) is a class of feedforward artificial neural network. It consists of at least three layers of nodes: an input layer, one or more hidden layers, and an output layer. Each node (neuron) in one layer connects to every node in the next layer (fully connected), and each connection carries a weight. A bias term is added to each neuron's computation. The network applies a non-linear activation function — here, sigmoid — to allow it to learn complex, non-linear decision boundaries that a single layer could never learn.
PURPOSE
Why do we need an MLP?
Single-layer networks (perceptrons) can only separate data with a straight line — they fail on XOR and most real problems. MLPs add hidden layers that transform the input space so that complex patterns become linearly separable by the time they reach the output neuron. Each layer learns increasingly abstract representations.
PARAMETERS
Key variables involved
w — weight on each connection
b — bias added per neuron
η — learning rate (step size)
σ(z) — sigmoid activation function
L — binary cross-entropy loss
δ — error gradient (delta)
Step-by-step explanation
Step 1 — Forward Pass
Inputs flow left to right. Each neuron computes a weighted sum of its inputs plus a bias, then squashes the result through the sigmoid function. Activations propagate layer by layer until the output neuron produces a prediction ŷ ∈ (0, 1).
z = Σ(wᵢ · xᵢ) + b
ŷ = σ(z) = 1 / (1 + e−z)
Step 2 — Compute Loss
The prediction ŷ is compared to the true label y using Binary Cross-Entropy loss. A high loss means the network was very wrong; the goal of training is to drive this number as close to 0 as possible across all training samples.
L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
Step 3 — Backpropagation
Using the chain rule, gradients of the loss with respect to every weight are computed, flowing backward through the network. Each neuron's delta tells us how much it contributed to the error — brighter nodes = bigger gradient.
δ_out = (ŷ−y)·ŷ·(1−ŷ)
δ_h = (Σ w·δ_next)·a·(1−a)
Step 4 — Weight Update
Every weight is nudged in the direction that decreases the loss, scaled by the learning rate η. This is Stochastic Gradient Descent (SGD). Smaller η → stable but slow; larger η → fast but may overshoot and never converge.
w ← w − η · ∂L/∂w
Step 5 — Repeat
Steps 1–4 repeat for every training sample, over many epochs. Gradually, weights converge to values that correctly classify all inputs. For XOR with this architecture, convergence typically happens within 200–500 epochs at η=0.10.
Epoch = one full pass
over all training samples
Small example (XOR)
Input [0,1]: forward pass → hidden neurons fire → output = 0.87 → loss is low → backprop finds small gradients → weights barely change. Input [0,0]: output = 0.12 → correct (XOR=0). After training, all 4 XOR cases are correct.
XOR([0,1]) = 1 → ŷ=0.87
XOR([1,1]) = 0 → ŷ=0.09

Navigation Guide

How to use the
Visualizer

Follow these steps to get the most out of the MLP Training Visualizer. Everything happens live in your browser — no setup needed.

Step-by-step guide
1
Open the Visualizer tab
Click the Visualizer button in the top navigation bar. You'll see the MLP network diagram, a loss curve panel, controls sidebar, and a dataset panel — all live and interactive.
2
Choose a training phase
At the top of the network panel you'll find three phase buttons — → Forward pass, ← Backprop, and ↑ Weight update. Click each to see a different stage of training highlighted in the network. Each recolours edges and nodes to show what's happening in that phase.
3
Load a dataset
In the bottom-left panel, you can drag & drop a CSV file, click to browse for one, or paste data manually into the text area. The built-in XOR dataset is pre-loaded — perfect for getting started. Format: each row is comma-separated values, with the last column as the label (0 or 1).
4
Adjust the controls
In the right sidebar, use the Learning Rate slider to control how large each weight update step is (try 0.05–0.30). Use the Speed slider to control how many training steps happen per animation frame. Higher speed = faster but harder to observe individual steps.
5
Start training
Press the Train button to begin. Watch the epoch counter climb, the loss value drop, and the accuracy rise. The loss curve below the network visualizes convergence in real time. Press ⏸ Pause at any time, and Reset to re-randomise weights and start over.
6
Read the colour legend
Blue edges = positive weights / forward signal strength. Red edges = negative weights / error gradients in backprop. Green edges = updated weights (thicker = higher magnitude). Purple nodes = neuron activation (brighter = more active).
7
Download your results
After training, use the Download Logs panel to export your training history. CSV gives you a spreadsheet-ready epoch log. JSON includes architecture metadata. Report generates a plain-text training summary with best-loss epoch highlighted.
Colour quick-reference
Blue — Forward Signal
Blue edges show positive weights and forward-pass activations. Brighter and thicker = stronger signal flowing through that connection during the forward phase.
Red — Error Gradient
Red edges show negative weights and error gradients during backprop. Brighter red nodes = that neuron received a larger share of the blame for the prediction error.
Green — Updated Weights
Green edges in Weight Update mode show final updated weights. Thicker line = higher weight magnitude. Reset to re-randomise and compare how different initialisations converge.
Quick tip
Set Speed to 1× and Learning Rate to 0.10 when learning. Once you understand what's happening, crank speed up to watch the loss converge fast. Hit Reset to see how weight initialisation affects convergence speed.
